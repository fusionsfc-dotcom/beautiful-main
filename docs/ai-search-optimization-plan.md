**작성일**: 2026년 4월 9일
**현황**: 기초 SEO 구축 완료, AI 검색 최적화는 미흡

## 1. 현황 진단

### 잘 되어 있는 것

- Open Graph / Twitter Card (소셜 공유 대응)
- 기본 Hospital 스키마 마크업
- 네이버 서치어드바이저 인증
- robots.txt + sitemap.xml 존재
- 주요 페이지별 meta title/description

### 핵심 문제점

| 문제 | 영향 | 심각도 |
|------|------|--------|
| SPA(Client-Side Rendering) 구조 | AI 크롤러가 페이지 내용을 읽지 못할 수 있음 | 높음 |
| 동적 페이지(클리닉 상세, 치료사례)가 sitemap에 없음 | 핵심 서비스 페이지가 AI에 비노출 | 높음 |
| 의사/서비스/FAQ 구조화 데이터 없음 | AI가 병원 정보를 구조적으로 파악 불가 | 높음 |
| 콘텐츠 깊이 부족 | AI가 답변 소스로 선택하지 않음 | 중간 |
| 리뷰/신뢰지표 구조화 데이터 없음 | E-E-A-T 점수 낮음 | 중간 |

## 2. AI 검색이란 무엇인가 (전략적 이해)

ChatGPT, Perplexity, Google AI Overview, 네이버 AI 등 AI 검색은 기존 검색과 다르게 작동합니다.

```
기존 검색: 사용자 → 키워드 → 링크 목록 → 사용자가 클릭
AI 검색:   사용자 → 질문 → AI가 웹을 읽고 → 답변 자체를 생성
```

즉 **"링크를 클릭하게 만드는 것"** 이 아니라
**"AI가 답변을 작성할 때 우리 병원 정보를 인용하게 만드는 것"** 이 목표입니다.

AI가 답변 소스로 선택하는 콘텐츠 조건:

1. 기계가 읽기 쉬운 구조 (JSON-LD 스키마)
2. 명확한 사실 정보 (누가, 무엇을, 어떻게)
3. 전문성 신호 (의사 자격, 치료 실적, 인용 가능한 수치)
4. 크롤링 가능성 (SPA 렌더링 문제 해결)

## 3. 개선 전략 (3단계)

### Phase 1: 크롤링 가능하게 만들기 (기반 작업)

**문제**: React SPA는 JavaScript가 실행되어야 내용이 보입니다. 많은 AI 크롤러(특히 Perplexity, ChatGPT)는 JS 실행 없이 HTML만 읽어갑니다. 현재 병원 소개, 클리닉 상세, 치료사례 페이지 내용이 이런 크롤러에게는 빈 화면입니다.

**해결 방향**:

- SSR(서버사이드 렌더링) 또는 정적 생성(Pre-rendering) 도입
- 핵심 콘텐츠를 HTML에 직접 포함
- sitemap.xml을 동적으로 생성 (클리닉 3개 상세 + 치료사례 포함)

**sitemap 추가 대상**:

- `/clinics/cancer` (암 치료 클리닉)
- `/clinics/neuro` (신경계 클리닉)
- `/clinics/pain` (척추·관절 클리닉)
- `/cases` 및 개별 사례 URL
- `/health-check`, `/reservation`

---

### Phase 2: AI가 읽을 수 있는 구조화 데이터 구축

현재 Hospital 스키마 1개만 있습니다. AI가 답변에 활용할 수 있도록 다음 스키마를 추가해야 합니다.

#### 추가할 스키마 목록

**① 의료진 스키마 (Doctor/Person)**

```
목적: "파주 한방병원 암 전문의"로 AI 검색 시 의사 정보 인용
내용: 의사 이름, 전문 분야, 자격증, 경력, 논문/방송 출연
```

**② 서비스 스키마 (MedicalTherapy / Service)**

```
목적: "항암 후 한방 치료", "이명 한방 치료" 검색 시 클리닉 노출
내용: 클리닉명, 대상 질환, 치료 방법, 소요 기간, 가격대
```

**③ FAQ 스키마 (FAQPage)**

```
목적: Google AI Overview에 FAQ 직접 노출
내용: 칼럼 페이지의 FAQ 섹션을 구조화
예시 Q: "항암 치료 중 한방 치료를 받을 수 있나요?"
```

**④ 아티클 스키마 (MedicalWebPage / BlogPosting)**

```
목적: 치료사례, 칼럼이 AI 답변 소스로 인용되게
내용: 제목, 작성일, 저자(의사), 카테고리, 요약
```

**⑤ 리뷰/평점 스키마 (AggregateRating)**

```
목적: 검색 결과에 별점 표시, 신뢰도 상승
내용: 환자 후기 기반 평균 평점, 리뷰 수
```

**⑥ 위치정보 스키마 강화 (LocalBusiness)**

```
목적: "파주 한방병원", "국립암센터 근처 한방병원" 지역 검색
내용: 정확한 주소, 영업시간, 주차, 대중교통
```

### Phase 3: AI가 인용하고 싶은 콘텐츠 만들기

스키마는 형식이고, 실제 인용은 콘텐츠 품질로 결정됩니다.

#### 3-1. 콘텐츠 전략: AI 답변용 정보 페이지

AI 검색에서 가장 자주 나오는 질문 유형에 맞춘 콘텐츠 제작:

| AI 검색 질문 예시 | 목표 콘텐츠 |
|---|---|
| "항암 치료 후 한방 치료 효과 있나요?" | 암 클리닉 상세 페이지 + 실제 사례 |
| "파주 이명 치료 잘하는 한방병원" | 이명 클리닉 전문 페이지 |
| "한방병원 입원 어떻게 되나요?" | 시설/입원 안내 구체적 설명 |
| "중풍 후유증 한방 재활 방법" | 중풍 클리닉 + 재활 칼럼 |
| "국립암센터 근처 한방병원 추천" | 위치 기반 About 페이지 강화 |

#### 3-2. 각 페이지에 추가해야 할 필수 정보

**클리닉 상세 페이지** (현재 부족한 것):

- 담당 의사 명시 (이름 + 자격 + 경력)
- 치료 과정 단계별 설명 (AI가 "어떻게 치료하나요"에 답할 수 있도록)
- 치료 기간/횟수 구체적 명시
- 대상 환자 기준 명시 (AI가 적합성 판단에 활용)

**치료사례 페이지** (현재 부족한 것):

- 각 사례에 진단명, 치료 기간, 치료 방법, 결과 수치화
- 담당 의사 연결
- 발행일, 카테고리 명시

**병원소개 페이지** (강화 필요):

- 의료진 개인별 전문 분야, 자격, 논문/방송 경력 상세화
- 병원 설립 연도, 누적 환자 수, 주요 실적 수치

#### 3-3. E-E-A-T 강화 (Experience, Expertise, Authoritativeness, Trustworthiness)

Google AI Overview와 ChatGPT는 의료 분야에서 신뢰도를 특히 중요하게 봅니다.

- **의사 저자 표시**: 모든 칼럼/사례에 "작성: OOO 원장" 명시
- **자격 증명 링크**: 한의사협회, 전문학회 링크
- **미디어 노출**: 방송/신문 인터뷰 기록을 페이지에 포함
- **학술 근거**: 치료법에 대한 근거 논문 참고 표시

## 5. 우선순위 실행 계획

### 즉시 (1-2주)

1. sitemap.xml에 클리닉 상세 3개 + 치료사례 추가
2. Bing Webmaster Tools 등록 (ChatGPT Search 대응)
3. Google Search Console에서 인덱싱 현황 확인
4. 카카오맵, 네이버 지도 정보 최신화

### 단기 (1달)

5. 의료진 스키마 추가 (About 페이지)
6. FAQ 스키마 추가 (칼럼 페이지)
7. 서비스 스키마 추가 (클리닉 3개)
8. SPA 프리렌더링 도입 또는 핵심 페이지 정적화

### 중기 (2-3달)

9. 치료사례 페이지 콘텐츠 고도화 (수치, 담당의 연결)
10. AI 검색 타겟 칼럼 10편 이상 제작
11. 리뷰/평점 스키마 추가
12. 의료진 개인 프로필 페이지 신설 검토
